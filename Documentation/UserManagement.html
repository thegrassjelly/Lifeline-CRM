User Management


The user management module includes a span of user centric management features available to administrators to further manage system users and increase productivity by better management and monitoring


The user master list includes a list of all users, spanning from guest, pending clients, clients, and administrators. The list includes a feature rich table which enables administrators to sort, filter, search, group and view users in the way convenient to them which will be discussed in the detail next.



An overview of the table header shows a input boxes which administrators can filter text or numeric information spanning from user ID's, names to membership types. Clicking the header name enables to sort the records both ascending and descending.



The table allows grouping of the records as seen above. By dragging the header name in the portion between the search bar and header bar, the records can be grouped as seen in the above capture.


After the administrator have decided on which record to pick, clicking the small arrow on the first column enables the administrator to get a detailed view of the record, most specifically the user.


As seen above, the main row has includes a brief summary of the record including a user photo, if one has uploaded any, name, user type, priority,

date registered/added, user status and a update view button the far right.


After expanding the view, the administrator is represented by a summary of user information such as employer, if the user is belonging to any employer/corporate renewal memberships, Medical history, secondary contacts, user sent messages, and membership renewals.



Clicking the rightmost button on the table, the administrator is redirected to the specific user detail, they have access to the users information, secondary contacts, medical information, add employer code, user sent messages and membership renewals by that user.


The administrator has control on changing the user information if necessary, add the medical information which are seen by the users in their respective accounts.




Similar to the first table, clicking the rightmost column named Message details, the administrators are redirected to the specific message detail such as the complete transcript of the message.




Clicking to a submitted renewal by the user, the administrator is then redirected to the specific renewal detail, where they have control on updating the detail of the renewal such as Payment being paid and unpaid, and Membership status to active and inactive.





Below the billing and payment information regarding the renewal details, is the uploaded deposit/payment slip if there are any uploaded by the user. This is where the administrators or customer care representatives review and verify the submitted payment/deposit slip which then after the administrator can activate the renewal for the users account. This will then reflect the users dashboard as discussed in the earlier parts of this documentation.



A control named Void Renewal is seen below the renewal details which are accessible by the administrators, let them to nullify the sent renewal if approved by the company.



A control the user master list page is the reports printing, enabling the administrator to print the list of users, dynamically updated if there are any new users in every new exportation of the reports. The reports has functions of exporting and printing the reports.



Other than registration by users, administrators can manually add users, which also includes adding specific user information as seen in the capture above.
